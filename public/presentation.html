What Contexta does

Problem
Product Managers think in frameworks and methodologies
Triggered when: PM faces a new problem
  • PM faces a new challenge and has to make a decision
  • PM has no one to ask for help
  • PM knows the problem was solved many times before
  • PM is stuck and creates decision paralysis
Solution
Contexta finds the right artifacts for any problem
Triggered when: PM discovers Contexta
  • PM provides context about the role, company stage, team size
  • PM describes the challenge in a few sentences
  • Contexta finds relevant artifacts based on the context and the challenge
  • Contexta displays artifacts with a short explanation
  • Contexta shows real-world content-pieces about the artifacts

MVP Showcase

  1. First time user starts on home page
  2. Starts Challenge flow and enters Context
  3. Defines the Challenge
  4. Contexta finds relevant Artifacts
  5. Artifacts are displayed with a short explanation
  6. Real-world content-pieces about the Artifacts are displayed
  7. Sign-up and save to Journey
  8. User can manage the Challenges to their personal library

Initial Project Setup

  1. Init Prompt: brief description of the project and the problem I want to solve
  2. Tech Stack: provided the tech stack of the project
  3. Requirements folder: setup the requirements folder with the first set of requirements for the project (PRDs)
  4. Claude.md: let it initialize the project with Claude.md file
  5. /skills: defined the skills for the project (/dev, /documentation, /code review)

My Workflow

  1. main agent: reads the requirements (PRD) and creates an epic in /specs folder
  2. /dev agent: reads the epic, implements the epic and runs unit tests and e2e tests
  3. /documentation agent: two modes to write the documentation. --all or --changed-files for the changed files only
  4. /code review agent: gets started manually by the user to review the code base
