What Contexta does

Product Managers tend to think in frameworks and methodologies
They are always looking for the right Product Management Artifact to solve a specific problem.

--> Contexta as an AI copilot helps Product Managers find the right Artifactsfor any problem.

Initial Project Setup

  1. Init Prompt: brief description of the project and the problem I want to solve
  2. Tech Stack: provided the tech stack of the project
  3. Claude.md: let it initialize the project with Claude.md file
  4. Requirements folder: setup the requirements folder with the first set of requirements for the project (PRDs)
  5. /skills: defined the skills for the project (/dev, /documentation, /code review)

My Workflow

  1. main agent: reads the requirements (PRD) and creates an epic in /specs folder
  2. /dev agent: reads the epic, implements the epic and runs unit tests and e2e tests
  3. /documentation agent: two modes to write the documentation. --all or --changed-files for the changed files
  4. /code review agent: gets started manually by the user to review the code base

MVP Showcase

  1. First time user starts on home page
  2. Starts flow and enters Context
  3. Defines the Chalennge
  4. Artifacts are recommended
  5. Artifacts Details are displayed
  6. Sign-up and save to playbook is displayed
