You are a senior full-stack architect and product engineer.

We are building the MVP of "Contexta".

Contexta is an AI-powered product management knowledge system.
It helps product managers describe a real challenge and receive structured, context-aware insights derived from curated content (podcasts, articles, frameworks, playbooks, thought leaders).

Your task:
Initialize a production-oriented MVP codebase using a clean, scalable Next.js monolith architecture with Supabase.

---------------------------------
# 1. Tech Stack (fixed)

Framework:
- Next.js (App Router, full-stack usage)
- TypeScript
- TailwindCSS

Backend logic:
- Next.js Route Handlers (/app/api)
- Supabase (Postgres, Auth, Storage)
- pgvector (for embeddings)
- OpenAI (initial LLM provider, swappable)
- Zod for validation

Infrastructure:
- Single deployable Next.js app
- Supabase as primary database
- No Express
- No microservices

---------------------------------
# 2. Architectural Principles

Follow these rules strictly:

1. Business logic must NOT depend directly on OpenAI.
   Create an AI Provider interface:
     - generateText()
     - generateEmbedding()
   Implement OpenAIProvider as the first provider.

2. Route handlers must remain thin.
   No business logic inside /api routes.
   All logic must live in service modules.

3. Data layer must be modular.
   Separate repositories for:
     - users
     - challenges
     - content
     - embeddings

4. Ingestion pipeline must be async-ready.
   Structure code so that it can later support background jobs.
   For MVP it may run synchronously.

5. Prompts must be centralized in a /core/prompts folder.

6. Keep architecture modular but within one Next.js monolith.

---------------------------------
# 3. MVP Core Flow (Activation Flow)

Implement this flow end-to-end:

1) User signs up via Supabase Auth

2) User creates profile:
   - role
   - company stage
   - team size
   - experience level

3) User submits a Challenge:
   - raw description (text)
   - challenge domain (strategy, discovery, delivery, growth, leadership)
   - optional subdomain
   - impact & reach

4) System:
   - generates challenge summary (LLM)
   - generates embedding
   - performs vector similarity search
   - retrieves top content chunks
   - generates structured recommendations

5) User sees:
   - challenge summary
   - top content matches
   - structured insights
   - suggested thought leaders

---------------------------------
# 4. Important Constraints

- Use dependency injection where reasonable.
- Prepare for future multi-provider AI support.
- Use async/await properly.
- Include structured error handling.
- Add logging hooks for future observability.
- Keep the system understandable for a new engineer.
- Do NOT overengineer.

---------------------------------

This is a production-minded but lean MVP.
Keep it clean, readable, and extensible.
Do not introduce unnecessary infrastructure.
